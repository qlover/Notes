<h3>贪婪</h3>
<p>贪婪模式也是默认的模式</p>

'12345678'.replace(/\d{3,6}/g, 'x')  // x78

<p>因为正则表达式一旦匹配成功就会自己会尽可能多的在匹配，也让就是说会匹配最多的一次，所以会返回 123456</p>

<h3>非贪婪</h3>
<p>非贪婪也就是只要匹配成功了就会停止匹配，而做法也非常简单，就是在量词后加上 ? 就行</p>

'12345678'.replace(/\d{3,6}?/g, 'x')  // xx78

<u>当匹配到了 123 这一次就会停止匹配，进行下一次匹配，下一次匹配到了 456 又会停止，直到到了 78 不匹配就停止所有匹配操作</u>
<p>由此就可以看出来贪婪与非贪婪的关系，</p>




<h2>前瞻</h2>
<ul>
<li>正则表达式从文本头部向尾部开始解析，文本尾部方向称为 “前”</li>
<li>前瞻就是在正则表达式匹配到规则的时候，向前检查是否符合断言，断言也就相当于条件一样，后顾/后瞻方向相反</li>
<li>javascript 不支持后顾</li>
<li>符合和不符合特定断言称为肯定/正向匹配和否定/负向匹配</li>
</ul>

</xmp>

<p>openId 是标识一微信用户的唯一标记，而上次群发接口时，就用上了 openId,

<p>第二个则是获取详细的用户信息，比如：昵称，头像，国籍</p>

</li>
<li>通过 code 换取网页授权 access_token

## 动态磁盘卷 ##

1.简单卷：大小可以增加不能减少
2.跨区卷：由多个物理的空间组成，没有容错功能，成员大小可不一样，当成员磁盘中任何一个发生故障，整个跨区卷数据都将全部丢失
3.带区卷(RAID-0)：由多个物理的空间组成，成员大小必须一样，没有容错功能，平均写入磁盘中,以 64KB 为单位
4.镜像卷(RAID-1)：两个不同物理空间,容错,利用率较低
